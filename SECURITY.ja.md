# セキュリティポリシー

ONESHIMのRustクライアントのセキュリティを真剣に受け止めています。脆弱性を発見された場合は、本ドキュメントに記載された手順に従って報告をお願いいたします。

## セキュリティ脆弱性の報告

**セキュリティ脆弱性を公開Issueとして報告しないでください。** 以下の非公開チャネルをご利用ください。

### 報告方法

1. **メール**: `security@oneshim.dev`宛にメールをお送りください。可能であればPGP暗号化をご使用ください。
2. **GitHub Security Advisory**: リポジトリの「Security」タブにある「Report a vulnerability」を選択して非公開で報告できます。

### 報告に含めていただきたい情報

効果的な対応のため、以下の情報をできる限り含めてください。

- **脆弱性の種類**: 該当する場合はCWE識別子（例: CWE-79 XSS、CWE-89 SQLインジェクション、CWE-200 情報漏洩）
- **影響を受けるクレイト**: 脆弱性が含まれるクレイト名とソースファイルパス（例: `crates/oneshim-vision/src/privacy.rs`）
- **再現手順**: 脆弱性を再現するためのステップバイステップの手順
- **影響範囲**: 脆弱性が悪用された場合に想定される影響（ローカルデータの漏洩、リモートコード実行など）
- **概念実証（PoC）**: 脆弱性を実証するコードやスクリーンショット（ある場合）
- **修正案**: 脆弱性の修正に関するアイデア（任意）
- **環境**: OS、Rustバージョン（`rustc --version`）、cargoバージョン、関連するクレイトのバージョン

### 特に重要なセキュリティ領域

ONESHIMのRustクライアントにおいて、以下の領域はセキュリティ上特に重要です。

- **スクリーンキャプチャとPIIフィルター**（`oneshim-vision`）: 画面上の個人識別情報のマスキング回避
- **ローカルSQLiteストレージ**（`oneshim-storage`）: 暗号化されていないデータへの不正アクセス
- **JWT認証トークン**（`oneshim-network`）: トークンの窃取または検証のバイパス
- **自動化コントロール**（`oneshim-automation`）: ポリシー検証のバイパスによる任意コマンド実行
- **自動アップデート**（`oneshim-app`）: アップデートバイナリの整合性検証のバイパス
- **ローカルWebダッシュボード**（`oneshim-web`）: ローカルAPIへの不正アクセス（`allow_external`設定に関連）

## サポート対象バージョン

以下のバージョンに対してセキュリティアップデートを提供します。

| バージョン | サポート状況 |
|---------|---------------|
| 最新の`main`ブランチ | サポート対象 |
| 最新のリリースタグ | サポート対象 |
| 過去のリリース | サポート対象外 |

正式リリースがまだ行われていないため、セキュリティ脆弱性は**最新の`main`ブランチ**に対して報告してください。

## 対応時間SLA

セキュリティ報告を受領した場合、以下のタイムラインに従って対応することを目指します。

| フェーズ | 目標タイムライン |
|-------|----------------|
| 受領確認 | 3営業日以内 |
| 脆弱性評価と対応計画 | 14日以内 |
| パッチリリース | 90日以内 |
| 報告者への通知と開示スケジュール | パッチリリース直後 |

緊急のセキュリティ問題（リモートコード実行や完全な認証バイパスなどの深刻度の高い脆弱性）については、優先対応を受けるためメールの件名に`[URGENT]`を含めてください。

## 責任ある開示ポリシー

ONESHIMのRustクライアントは**責任ある開示（Responsible Disclosure）**ポリシーに従います。

### 私たちのコミットメント

- 報告者のプライバシーを保護します。
- 修正完了時に報告者に通知し、報告者の同意のもとで開示スケジュールを調整します。
- 報告者の貢献をSecurity Advisoryでクレジットします（ご希望の場合）。
- 善意のセキュリティリサーチ活動に対して法的措置を取ることはいたしません。

### 報告者へのお願い

- 脆弱性が修正されるまで公開を控えてください。
- 脆弱性の検証が他のユーザーのデータやサービスに影響を与えないようにしてください。
- 事前の承認なくデータの破壊、変更、持ち出しを行わないでください。
