# v0.4 Extended Features Spec

**Date**: 2026-04-05
**Scope**: Time-series stats, Audit log viewer, Policy configuration UI
**Prerequisite**: Phase 1-4 + Batch A complete

## 1. Problem Statement

| # | Feature | Infrastructure | Gap |
|---|---------|---------------|-----|
| 1 | Time-series stats | SQLite `suggestions` table with `created_at`, `suggestion_type`, `source`, `acted_at` columns + indexes | No aggregation queries, no time-bucketed API, no chart UI |
| 2 | Audit log viewer | `AuditLogger` with VecDeque buffer (1000 max) + REST endpoints (`/automation/audit`) + `AuditQueryService` | No web dashboard page to display logs |
| 3 | Policy config UI | `ExecutionPolicy` struct + `PolicyClient` + `PolicyCache` + read-only `GET /automation/policies` | No CRUD endpoints, no editing UI, no persistence |

## 2. Design

### 2.1 Time-Series Stats

**Approach**: Query SQLite suggestions table directly with time-bucketed aggregation. No new tables needed — existing schema has all required columns.

#### 2.1.1 Storage query methods

**File**: `crates/oneshim-storage/src/sqlite/edge_intelligence/suggestions.rs`

Add to `SqliteStorage`:

```rust
/// Daily suggestion stats for the last N days.
pub fn suggestion_daily_stats(&self, days: u32) -> Result<Vec<DailyStatRecord>, StorageError> {
    // NOTE: created_at is RFC3339 text, use SUBSTR for day extraction
    // SELECT SUBSTR(created_at, 1, 10) as day, COUNT(*) as total,
    //        SUM(CASE WHEN acted_at IS NOT NULL THEN 1 ELSE 0 END) as acted,
    //        suggestion_type, source
    // FROM suggestions
    // WHERE created_at >= datetime('now', '-N days')
    // GROUP BY day, suggestion_type, source
    // ORDER BY day DESC
}
```

**Return type** in `crates/oneshim-core/src/models/storage_records.rs`:

```rust
pub struct DailyStatRecord {
    pub day: String,           // "2026-04-05"
    pub total: u32,
    pub acted: u32,
    pub suggestion_type: String,
    pub source: String,
}
```

#### 2.1.2 IPC command

**File**: `src-tauri/src/commands/suggestions.rs`

```rust
#[command]
pub async fn get_suggestion_daily_stats(
    app_state: tauri::State<'_, AppState>,
    days: Option<u32>,
) -> Result<Vec<DailyStatDto>, String>
```

#### 2.1.3 Frontend chart

**File**: `crates/oneshim-web/frontend/src/overlay/components/SuggestionStats.tsx`

Add a "Daily Trends" section with a simple bar chart showing daily totals for the last 7 days. Uses the existing stats tab layout.

### 2.2 Audit Log Viewer

**Approach**: The REST endpoint `GET /automation/audit` already returns audit entries. Build a web dashboard page that consumes it.

#### 2.2.1 Web dashboard page

**File**: `crates/oneshim-web/frontend/src/pages/audit/index.tsx` (NEW)

Features:
- Table view of audit entries (entry_id, timestamp, action, status, details, elapsed_ms)
- Status filter (Started/Completed/Failed/Denied/Timeout)
- Auto-refresh on interval (10 seconds)
- Status badge colors: Completed=green, Failed=red, Denied=orange, Timeout=yellow, Started=blue

**File**: `crates/oneshim-web/frontend/src/App.tsx`
- Add route: `<Route path="/audit" element={<AuditLog />} />`

#### 2.2.2 No backend changes needed

REST endpoint already exists and works:
- `GET /automation/audit?limit=100&status=Failed` → `Vec<AuditEntryDto>`
- `GET /automation/policy-events?limit=100` → `Vec<AuditEntryDto>`
- `GET /automation/stats` → `AutomationStatsDto`

### 2.3 Policy Configuration UI

**Approach**: Add CRUD REST endpoints + web dashboard page for managing policies. Persist to config file via existing ConfigManager.

#### 2.3.1 CRUD REST endpoints

**File**: `crates/oneshim-web/src/handlers/automation/execution.rs`

Add handlers:
- `POST /automation/policies` → create new ExecutionPolicy
- `PUT /automation/policies/{id}` → update policy
- `DELETE /automation/policies/{id}` → delete policy

Each handler calls `PolicyClient::update_policies()` with the modified policy list and persists via ConfigManager.

**File**: `crates/oneshim-web/src/routes.rs`
- Register new routes

#### 2.3.2 Policy persistence

**File**: `crates/oneshim-automation/src/policy/mod.rs`

Add `PolicyClient` methods:
- `pub async fn add_policy(&self, policy: ExecutionPolicy)` → validates + adds to cache
- `pub async fn remove_policy(&self, policy_id: &str) -> bool` → removes from cache
- `pub async fn list_policies(&self) -> Vec<ExecutionPolicy>` → returns current cache

Persist to JSON config file via ConfigManager after each mutation.

**Config field addition**: Add `pub policies: Vec<ExecutionPolicy>` to `AutomationConfig` in `crates/oneshim-core/src/config/sections/privacy.rs` (where `AutomationConfig` lives). Default: empty vec.

#### 2.3.3 Web dashboard page

**File**: `crates/oneshim-web/frontend/src/pages/policies/index.tsx` (NEW)

Features:
- Table of existing policies (process_name, confirmation, audit_level, sandbox)
- "Add Policy" button → form modal
- Edit/Delete actions per row
- Form fields: process_name, confirmation (Auto/Confirm/Block), audit_level, sandbox_profile, allowed_args, max_execution_time_ms

## 4. Non-Goals

- Persistent audit log to SQLite (in-memory buffer is sufficient for v0.4)
- Time-series charts with granularity finer than daily
- Role-based policy access control
