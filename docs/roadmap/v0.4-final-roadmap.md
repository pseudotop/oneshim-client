## Phase 1 — Suggestion Real-World UX

Priority: highest. Makes the core value proposition usable.

### 1.1 SSE Reconnection
- `SuggestionReceiver::run()` currently breaks on `SseEvent::Close` with no retry
- Add exponential backoff reconnect loop (1s → 30s, matching `SseStreamClient` pattern)
- Preserve queue state across reconnections
- **Files:** `crates/oneshim-suggestion/src/receiver.rs`, `src-tauri/src/scheduler/loops/suggestions.rs`

### 1.2 New Suggestion Notification
- Badge count on overlay icon when new suggestions arrive
- Desktop notification via `DesktopNotifier::show_suggestion()`
- Tauri event `overlay:suggestion-count-changed` for UI badge update
- **Files:** `crates/oneshim-suggestion/src/receiver.rs`, overlay frontend components

### 1.3 History UI
- Add "History" tab to SuggestionsPanel
- Show accepted/rejected/deferred with timestamps
- `get_suggestion_history` IPC already exists — just wire to UI
- **Files:** overlay frontend (new `SuggestionHistory.tsx` component)

### 1.4 Deferred Management
- Deferred suggestions re-surface after configurable delay (default: 2 hours)
- "Snooze" option with duration picker (30m, 1h, 2h, tomorrow)
- Timer-based re-queue in `SuggestionQueue`
- **Files:** `crates/oneshim-suggestion/src/queue.rs`, overlay frontend

### 1.5 Feedback Error Handling
- Show toast notification on feedback send failure
- Retry queue for failed feedback submissions
- Visual indicator (checkmark/X) on suggestion card after action
- **Files:** `crates/oneshim-suggestion/src/feedback.rs`, overlay frontend

## Phase 2 — Chat + Suggestion Integration

### 2.1 Chat-Initiated Suggestions
- Add "Get suggestions" button/command in chat UI
- Chat sends context-aware request to server
- Server response populates suggestion queue
- **Files:** chat page, new IPC command, server API

### 2.2 Suggestion Context in Chat
- Clicking a suggestion opens chat with that context pre-filled
- "Explain this suggestion" action on suggestion cards
- **Files:** overlay frontend, chat page

### 2.3 Chat Response → Suggestion Conversion
- Server can mark chat responses as "actionable suggestions"
- These auto-appear in the suggestion panel
- **Files:** chat backend, suggestion receiver

## Phase 3 — YELLOW Domain Stabilization
