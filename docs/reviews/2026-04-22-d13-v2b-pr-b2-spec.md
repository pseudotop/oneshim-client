## 2. Goals & Non-goals

### 2.1 Goals

1. `SubscribeMetrics` yields `SubscribeMetricsResponse` items. First yield is always a `ServerLoadHint`; subsequent yields are `MetricBucket` data interleaved with transition/heartbeat `ServerLoadHint`.
2. Two cadence modes:
   - **Realtime** (`interval_secs == 0`): wake on `event_tx::Metrics` tick, coalesce within 10ms quiet window, rate-limit by effective interval.
   - **Interval** (`interval_secs > 0`): fire on a `tokio::time::interval` ticker with `MissedTickBehavior::Skip` (IMP-10 — drift-free vs prior `sleep`).
3. Load classification (`LoadPolicy`) drives enforcement. Four levels (`Low` / `Medium` / `High` / `Critical`) mapped from CPU% + free-memory-GiB, with 30s warm-up forcing `Medium`.
4. **Effective trust policy (CRIT-6, D2)**: "every loopback caller is trusted to opt out of hints" — since v2b binds loopback only. The `honor_opt_out` function accepts a token-match path as forward-compat for v2c. Under PR-B2 runtime the token branch is dead code (reachable only in unit tests). Spec explicitly documents this truth.
5. Runtime kill switch: `WebConfig.grpc_streaming_enabled = false` → `subscribe_metrics` returns **`Status::unavailable("streaming disabled")`** (IMP-1, IMP-15 — not `Unimplemented` which carries permanent-feature-absent semantics; not a config-field-leaking message).
6. All 10 existing v2a integration tests continue to pass.
7. **Active-stream cap globally enforced** across both RPC surfaces (PR-B2 only SubscribeMetrics is live). Default cap = 50, configurable via `GrpcSpawnConfig.max_concurrent_streams` to enable cheap cap-breach tests (CRIT-12). Cap check happens BEFORE auth gate and BEFORE hint emission (CRIT-8, IMP-18).

### 2.2 Non-goals

- `SubscribeEvents` implementation (PR-B3)
- New proto messages/RPCs beyond PR-B1
- Cross-subscriber DB query coalescing (v3 backlog)
- TLS, dedicated gRPC auth token, external-bind (v2c)
- `Ping` event emission
- Storage-level changes (PR-B1 scope)
- Keystroke/click counter plumbing — v2a `grpc/mod.rs:258-259` hardcodes `active_keystrokes: 0, active_mouse_clicks: 0` because `SystemMetrics` has no such fields (IMP-19). **PR-B2 preserves this zero-output** for parity. Tracked as separate future work (requires adding typing_wpm-derived keystroke estimate OR extending `SystemMetrics`).

## 3. Wire contract (already-landed proto — no changes in PR-B2)

Quoted from `crates/oneshim-web/src/proto/generated/oneshim.dashboard.v1.rs` (generated in PR-B1):

```proto
rpc SubscribeMetrics(SubscribeMetricsRequest) returns (stream SubscribeMetricsResponse);

message SubscribeMetricsRequest {
  uint32 interval_secs        = 1;   // 0 = realtime, N = N-second bucket
  bool   respect_server_hints = 2;   // false = opt-out (trust-gated)
}

message SubscribeMetricsResponse {
  oneof payload {
    MetricBucket    data = 1;
    ServerLoadHint  hint = 2;
  }
}

message MetricBucket {                             // top-level, shared with v2a (promoted in PR-B1)
  google.protobuf.Timestamp start = 1;
  double cpu_avg_pct    = 2;
  double memory_avg_mb  = 3;
  uint32 active_keystrokes   = 4;                  // PR-B2 emits 0 (IMP-19)
  uint32 active_mouse_clicks = 5;                  // PR-B2 emits 0 (IMP-19)
}

message ServerLoadHint {
  enum Level {
    LOAD_LEVEL_UNSPECIFIED = 0;                    // CRIT-1: canonical names
    LOAD_LEVEL_LOW         = 1;
    LOAD_LEVEL_MEDIUM      = 2;
    LOAD_LEVEL_HIGH        = 3;
    LOAD_LEVEL_CRITICAL    = 4;
  }
  Level  load_level                 = 1;
  float  cpu_pct                    = 2;
  float  memory_pct                 = 3;
  uint32 suggested_interval_secs    = 4;
  uint32 suggested_event_rate_limit = 5;           // PR-B2 emits 0 (CRIT-2 — real value lands in PR-B3 when EventRateLimiter exists)
  string reason                     = 6;           // human-readable, incl. "warmup" prefix during first 30s (IMP-8)
  google.protobuf.Timestamp emitted_at = 7;
}
```

**V2a Rust clients migration note (IMP-2)**: PR-B1 commit c456367b promoted `productivity_metrics_response::MetricBucket` to top-level. Wire format unchanged (same field numbers + types + enclosing `repeated` tag). Rust clients update the type import path; no recompile-with-new-wire required. External non-Rust clients unaffected.

**Clamping & defaults (IMP-4 — removed nonsense row)**:

| Input | Server behavior |
|---|---|
| `interval_secs > 60` | Clamp to `INTERVAL_CEILING = 60s` |
| `interval_secs == 0` | Realtime mode (floor `INTERVAL_FLOOR = 250ms`) |
| `respect_server_hints = true` | Enforcement always on |
| `respect_server_hints = false` + loopback caller | Enforcement off (floor + ceiling still apply) |
| `respect_server_hints = false` + token match (forward-compat, dead code in v2b) | Enforcement off |
| `respect_server_hints = false` + no match | Treated as `true`, warn-log once per stream with `remote_is_loopback=false, auth_header_present=<bool>` (no token substring) |
| `grpc_streaming_enabled = false` | Immediate `Status::unavailable("streaming disabled")` (IMP-1, IMP-15) |
| Active stream cap exceeded | Immediate `Status::resource_exhausted("concurrent stream cap reached")` (CRIT-4, CRIT-8) |

### 4.1 `LoadPolicy`

Inputs:
- `thresholds: LoadThresholds` — `LoadThresholds { min_free_mem_gb, cpu_low_pct, cpu_medium_pct, cpu_high_pct }` with serde `#[serde(default = …)]`. Default `(2.0, 50.0, 70.0, 90.0)`.
- `started_at: Instant` — set by `::new()`; warm-up window = first 30s.

Outputs:
- `classify(metrics: &SystemMetrics) -> LoadLevel`:
  - First 30s → `Medium` regardless (avoid sysinfo stabilisation noise).
  - Else use `free_mem_gb = (memory_total.saturating_sub(memory_used)) as f32 / 1_073_741_824.0` (saturating_sub guards wrap — not panic, since f32 doesn't panic on subtraction; the semantic guard is against `memory_used > memory_total` returning nonsensical negatives after `as f32` cast (M3 nit caught)).
  - `Low` iff `cpu_pct < cpu_low_pct` AND `free_mem_gb > min_free_mem_gb * 1.5`
  - `Medium` iff `cpu_pct < cpu_medium_pct` AND `free_mem_gb > min_free_mem_gb`
  - `High` iff `cpu_pct < cpu_high_pct` AND `free_mem_gb > min_free_mem_gb * 0.5`
  - `Critical` otherwise (**disjunctive fallthrough** — M1 nit fix — either CPU or memory alone can escalate)
- `enforced_metrics_interval(level, requested_secs) -> Duration`:
  - `requested = if requested_secs == 0 { INTERVAL_FLOOR = 250ms } else { Duration::from_secs(requested_secs as u64) }`
  - `level_floor = match level { Low: 250ms, Medium: 1s, High: 5s, Critical: 30s }`
  - Return `requested.max(level_floor).min(INTERVAL_CEILING = 60s)`

Constructor validation: `LoadPolicy::new` uses `assert!` (NOT `debug_assert!` — M6 nit) that `cpu_low < cpu_medium < cpu_high` AND `cpu_high <= 100.0`.

### 4.2 `HintEmitter`

Per-stream state machine (comment in-code captures IMP-30 drop-ordering note: "`last_*` fields updated before `return Some(...)` so subsequent maybe_emit after a yield sees the updated state; the yield-before-update pattern used in earlier design drafts would race if the generator resumed after a panic"):

```
state = (last_level: Option<LoadLevel>, last_emit_at: Option<Instant>)

maybe_emit(level, policy, cpu_pct, memory_pct, is_warmup) -> Option<ServerLoadHint>:
  now = Instant::now()
  should_emit =
    last_level.is_none()                                 // first call
    OR last_level != Some(level)                         // transition
    OR last_emit_at.elapsed() >= HEARTBEAT (30s)         // heartbeat

  if !should_emit: return None
  last_level = Some(level); last_emit_at = Some(now)      // mutate BEFORE yield
  return Some(build_hint(level, policy, cpu_pct, memory_pct, is_warmup))

build_hint fields:
  load_level                 = ProtoLevel::{Low|Medium|High|Critical}   // NOTE: canonical enum variant names LOAD_LEVEL_* (CRIT-1)
  cpu_pct / memory_pct       = passed through
  suggested_interval_secs    = policy.enforced_metrics_interval(level, 0).as_secs() as u32
  suggested_event_rate_limit = 0                                         // CRIT-2: hardcoded for PR-B2. PR-B3 populates via EventRateLimiter
  reason                     = if is_warmup { format!("warmup MEDIUM (cpu={cpu:.1}% mem={mem:.1}%)") }   // IMP-8 restored
                               else { format!("{tag} (cpu={cpu:.1}% mem={mem:.1}%)") }                   // tag = "LOW" | "MEDIUM" | ...
  emitted_at                 = Utc::now() -> google.protobuf.Timestamp
```

The `is_warmup` bool is computed by `LoadPolicy::is_in_warmup(&self) -> bool` (new method, 2-line impl checking `started_at.elapsed() < WARMUP`). Exposed so HintEmitter can vary the reason without duplicating the clock.

### 4.3 `auth_gate::honor_opt_out` + `:authority` validator

**`honor_opt_out`** (pure function, returns `true` = enforcement must stay ON):

```rust
pub fn honor_opt_out(
    req_respect_hints: bool,
    remote_addr: Option<SocketAddr>,
    auth_header: Option<&str>,
    configured_token: Option<&str>,
) -> bool {
    if req_respect_hints { return true; }  // client opted in
    if let Some(addr) = remote_addr {
        if is_local_loopback(&addr.ip()) { return false; }  // trust(a): loopback — IPv6-mapped v4 covered (IMP-7)
    }
    // Trust(b) token match — forward-compat for v2c; under PR-B2 runtime this is dead code (loopback-only bind)
    let normalized_token = configured_token
        .and_then(|t| { let t = t.trim(); if t.is_empty() { None } else { Some(t) } });  // IMP-14 empty-token bypass fix
    if let (Some(h), Some(t)) = (auth_header, normalized_token) {
        // RFC 7235 + REST parity (IMP-13): case-insensitive "Bearer", trim, constant-time compare (IMP-11 / D9)
        if let Some(presented) = h.trim().strip_prefix_ignore_ascii_case("Bearer ").map(str::trim) {
            use subtle::ConstantTimeEq;
            if presented.as_bytes().ct_eq(t.as_bytes()).into() { return false; }
        }
    }
    true  // default: enforce
}

fn is_local_loopback(ip: &IpAddr) -> bool {
    match ip {
        IpAddr::V4(v4) => v4.is_loopback(),
        IpAddr::V6(v6) => v6.is_loopback()
            || v6.to_ipv4_mapped().map_or(false, |v4| v4.is_loopback()),  // IMP-7
    }
}
```

(`strip_prefix_ignore_ascii_case` is a small inline helper — 4 LoC — since stdlib doesn't provide it.)

**`:authority` validator** (IMP-12 / D10 DNS-rebinding protection):

```rust
pub fn validate_authority(authority: Option<&str>) -> Result<(), Status> {
    const ALLOWED: &[&str] = &["localhost", "127.0.0.1", "[::1]"];
    let host = authority
        .ok_or_else(|| Status::invalid_argument("missing :authority"))?
        .split(':').next().unwrap_or("");  // strip port
    if ALLOWED.iter().any(|a| {
        let a_host = a.trim_start_matches('[').trim_end_matches(']');
        a_host.eq_ignore_ascii_case(host) || a.eq_ignore_ascii_case(host)
    }) {
        Ok(())
    } else {
        Err(Status::permission_denied("authority not allowlisted"))
    }
}
```

Invoked by `subscribe_metrics` from `req.metadata().get("grpc-accept-encoding")`-style lookup of `:authority` (tonic/tower-exposed at the Request level via `req.uri().authority()`). Exact API verified in plan phase.

**Runtime opt-out semantics under PR-B2 (CRIT-6 / D2 truth)**: since v2b binds loopback only, **every production request short-circuits at the loopback branch**. The token-match branch exists solely as forward-compat; unit tests exercise it but no integration test can reach it until v2c removes loopback-only. Spec captures this so reviewers don't mistake coverage for live behavior.

**`tonic::Request::remote_addr()` availability (CRIT-7)**: Default tonic `Server::builder().add_service(...).serve(addr)` does NOT attach a `ConnectInfo` layer. Plan phase MUST run a smoke test verifying the value. If `None`, `serve()` gets extended with `tower::service_fn`-wrapped `TcpConnectInfo` — adds ~10 LoC. Unit test `auth_gate_handles_missing_remote_addr_as_enforce` pins the fallback (`None` → `true = enforce`, NOT granting trust).

### 4.6 `subscribe_metrics` generator (restructured — CRIT-3/4/5)

```text
subscribe_metrics(req):
  // Step 0a: authority validation (IMP-12 — before anything else)
  validate_authority(req.uri().authority().map(|a| a.as_str()))?

  // Step 0b: active-stream cap (CRIT-4 CAS-style with revert-on-over; IMP-18 global cap;
  //          CRIT-8 before auth + streaming_enabled + hint emission)
  let guard = StreamCounterGuard::try_acquire(active_streams.clone(), max_concurrent_streams)?
  // try_acquire impl: fetch_add(1); if prev >= cap { fetch_sub(1); return Err(Status::resource_exhausted(...)) }
  // Drop impl: fetch_sub(1). Captured by async_stream! closure — survives all yield-return paths (CRIT-3).

  // Step 0c: streaming kill switch (returns Status::unavailable not unimplemented — IMP-1, IMP-15)
  if !streaming_enabled: return Err(Status::unavailable("streaming disabled"))

  // Step 1: request parse + auth gate
  remote_addr = req.remote_addr()                 // may be None — CRIT-7 fallback
  auth_header = req.metadata().get("authorization").and_then(|v| v.to_str().ok()).map(String::from)
  { interval_secs, respect_server_hints } = req.into_inner()

  enforcement_on = honor_opt_out(respect_server_hints, remote_addr, auth_header.as_deref(), token.as_deref())
  if !respect_server_hints && enforcement_on:
    warn!(
      subscriber_id = %uuid::new_v4(),
      remote_is_loopback = remote_addr.map(|a| is_local_loopback(&a.ip())).unwrap_or(false),
      auth_header_present = auth_header.is_some(),
      "SubscribeMetrics opt-out rejected (untrusted)"
    )  // CRIT-9 — explicit field whitelist, NO #[tracing::instrument]

  let mut rx = event_tx.subscribe()
  let mut hint_emitter = HintEmitter::new()
  let mut last_emit: Option<Instant> = None
  let mut consecutive_db_failures: u32 = 0      // IMP-6 / D4 — counter for degraded-hint emission

  // Step 2: interval mode setup
  let mut ticker = None
  if interval_secs > 0:
    let eff = load_policy.enforced_metrics_interval(level_best_guess, interval_secs)
    ticker = Some(tokio::time::interval(eff).with_missed_tick_behavior(MissedTickBehavior::Skip))
    // IMP-10 — interval-based (drift-free) not sleep-based

  return async_stream! {
    // Move guard into closure so Drop decrements counter on generator termination.
    let _counter_guard = guard;
    loop {
      // A. Wait for next tick — before expensive ops (CRIT-5 restructure)
      if interval_secs == 0:
        match rx.recv().await:
          Ok(RealtimeEvent::Metrics(_)) => wake
          Ok(_)                          => continue   // non-metrics event
          Err(Lagged(_))                 => continue   // metrics tick will refire
          Err(Closed)                    => return
        // Coalesce 10ms quiet window
        while timeout(10ms, rx.recv()).await.is_ok() {}
        // Rate-limit BEFORE expensive work: skip if too soon since last_emit
        if let Some(t) = last_emit {
          if t.elapsed() < effective_interval_cache { continue }   // tight skip — no collect_metrics, no classify, no DB
        }
      else:
        ticker.as_mut().unwrap().tick().await

      // B. Classify + maybe emit hint
      metrics = match system_monitor.collect_metrics().await {
        Ok(m) => m,
        Err(e) => { warn!(err.code = %e.code(), "metrics_snapshot failed"); yield Err(Status::internal("metrics snapshot failed")); return; }
        // IMP-27 / D5 — exit on SystemMonitor failure (no 5s stale tolerance)
      }
      level = load_policy.classify(&metrics)
      is_warmup = load_policy.is_in_warmup()
      (cpu_pct, mem_pct) = derive_percents(&metrics)   // helper with 1 line each (M2 explicit)

      if let Some(h) = hint_emitter.maybe_emit(level, &load_policy, cpu_pct, mem_pct, is_warmup):
        yield Ok(hint(h))

      // C. Compute / refresh effective interval if level changed
      effective_interval_cache = if enforcement_on {
          load_policy.enforced_metrics_interval(level, interval_secs)
      } else {
          let r = if interval_secs == 0 { INTERVAL_FLOOR }
                  else { Duration::from_secs(interval_secs as u64) };
          r.max(INTERVAL_FLOOR).min(INTERVAL_CEILING)
      }
      if let Some(t) = ticker.as_mut() {
          if t.period() != effective_interval_cache {
              *ticker = Some(tokio::time::interval(effective_interval_cache)
                  .with_missed_tick_behavior(MissedTickBehavior::Skip));
          }
      }
      // NOTE (IMP-5, IMP-29): window_start uses effective_interval_cache (post-update), so
      // the FIRST bucket after a transition still uses previous interval's window. 1-tick
      // smearing. Acceptable; documented in §11.

      // D. Fetch bucket via spawn_blocking (IMP-20: aggregate_metrics_window is SYNCHRONOUS)
      window_start = Utc::now() - chrono::Duration::from_std(effective_interval_cache)
                                    .expect("effective_interval_cache bounded by INTERVAL_CEILING")   // IMP-9
      window_end = Utc::now()
      let storage_clone = storage.clone()
      match tokio::task::spawn_blocking(move || storage_clone.aggregate_metrics_window(window_start, window_end)).await {
        Ok(Ok(b)) => {
            consecutive_db_failures = 0;
            yield Ok(data(MetricBucket {
                start: Some(to_proto_ts(b.start)),
                cpu_avg_pct: b.cpu_avg_pct,
                memory_avg_mb: b.memory_avg_mb,
                active_keystrokes: 0,     // IMP-19: no source in SystemMetrics
                active_mouse_clicks: 0,   // IMP-19
            }));
            last_emit = Some(Instant::now());
        }
        Ok(Err(e)) => {
            consecutive_db_failures += 1;
            warn!(err.code = %e.code(), consecutive = consecutive_db_failures, "aggregate_metrics_window failed");
            // IMP-6 / D4: at N=5, force an extra hint with degraded reason
            if consecutive_db_failures == 5 {
                yield Ok(hint(build_hint_degraded("db_error_degraded")));
            }
            if consecutive_db_failures >= 10 {
                yield Err(Status::internal("persistent storage errors"));
                return;
            }
            continue;   // skip this tick, stream stays open
        }
        Err(join_err) => {
            yield Err(Status::internal("task join failure"));   // no task-id leak (M7)
            return;
        }
      }
    }
  }
```

**Key restructures vs iter-1**:
- CRIT-3: `_counter_guard = guard;` moved into `async_stream!` closure → `Drop` runs on every exit path (abrupt disconnect, JoinError, explicit return, `yield Err(...)` + return).
- CRIT-4: cap check uses revert-on-over CAS-style; `StreamCounterGuard::try_acquire` does `fetch_add → if prev >= cap { fetch_sub; Err }`.
- CRIT-5: in realtime mode the `last_emit.elapsed() < effective_interval_cache` check happens BEFORE `collect_metrics().await` — saves CPU on throttled iterations.
- CRIT-8: cap + authority + streaming_enabled checks all happen BEFORE any auth / hint work.
- IMP-10: `tokio::time::interval + MissedTickBehavior::Skip` (drift-free) replaces `sleep`.
- IMP-6 / D4: consecutive DB failure counter with degraded-hint + close-after-10.
- IMP-20: `spawn_blocking` wraps the **synchronous** `aggregate_metrics_window`.

## 6. Security & privacy posture for PR-B2

| Concern | PR-B2 posture |
|---|---|
| Wire confidentiality | Loopback-only bind inherited from v1/v2a (v2c flips the gate) — no network-level leak |
| PII on wire | `MetricBucket` fields are aggregated numerics; `ServerLoadHint` is server-authored numeric+enum+reason. **Zero PII paths in PR-B2 payloads.** Explicit DROP of `pii_sanitizer` forward-compat field (CRIT-15 / IMP-17) — PR-B3 introduces with wiring |
| Opt-out trust model | Loopback caller → trusted. Token path is forward-compat dead code in v2b runtime (loopback-only bind per CRIT-6 / D2). Unit tests exercise token path; integration tests exercise loopback path |
| Token comparison | **Constant-time** via `subtle::ConstantTimeEq` (IMP-11 / D9) to prevent timing-extraction even under DNS-rebind + `performance.now()` probes. Token normalization: `strip_prefix_ignore_ascii_case("Bearer ")` + `.trim()` matches REST at `lib.rs:524` (IMP-13); empty/whitespace tokens normalize to `None` (IMP-14) |
| `:authority` validation | **NEW tower layer** rejects DNS-rebound `Host`/`:authority` values (IMP-12 / D10). Allowlist: `localhost`, `127.0.0.1`, `[::1]` (case-insensitive) |
| Token leakage via logs | **Hard prohibition on `#[tracing::instrument]`** on `subscribe_metrics`, `honor_opt_out`, any function taking `auth_header` or `&GrpcSpawnConfig` (CRIT-9). Explicit whitelist of log fields: `subscriber_id`, `interval_secs`, `respect_hints`, `remote_is_loopback: bool`, `auth_header_present: bool`. CI grep-check (shell script in `scripts/` or `.github/workflows/`) enforces. `GrpcSpawnConfig` custom `Debug` redacts token (IMP-16) — test asserts `format!("{:?}", cfg).contains(token_value) == false` |
| DoS via many subscribers | Global `active_stream_counter: AtomicUsize` (default cap 50, test-configurable via `GrpcSpawnConfig.max_concurrent_streams`). Cap check BEFORE auth gate (CRIT-8, IMP-18) to fail unauth floods cheaply |
| Kill-switch response | `Status::unavailable("streaming disabled")` — neutral message (IMP-15). Test asserts message does NOT contain `"grpc_streaming_enabled"` or `"false"` substrings |
| IPv6-mapped v4 | `is_loopback` path handles `::ffff:127.0.0.1` via `to_ipv4_mapped()` + recursive check (IMP-7) |
